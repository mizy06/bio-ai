protocol_record、protocol_version、protocol_step、protocol_material、protocol_attachment、protocol_reference。

数据来源与覆盖领域

来自 protocols.io 官方平台，覆盖湿实验、干实验与跨学科 protocol；在湿实验侧尤其适合作为结构化步骤语料主源。

数据规模与更新情况

平台持续更新，公开页面 sitemap 可核约 158,352 条 protocol URL；同一 protocol 可能因版本或路由产生多个 URL。

下载单元数量

若按公开 protocol 页面 URL 口径统计，当前可核约 158,352 条。

单个下载单元平均大小

按前 5 条公开 protocol 页面 HTML 抽样计算，平均约 12.95 KB；不含附件。

数据格式与字段结构

核心对象可概括为 protocol → version → step，protocol → materials / reagents / equipment，protocol → attachments / media。

数据内容特点与适用场景

适合步骤抽取、材料规范化、protocol 生成、protocol 检索和 protocol 版本对齐。

获取方式、开放性与可用性

公开页面可访问；开发者 API 存在，但更深层访问依赖授权。

准入判断

公开页面抓取成功；受控 API 仍以授权访问为主。

最小爬取伪代码

1. 先读取 `robots.txt` 明确公开页面与 `/api/` 的访问边界。
2. 把 `developers` 页作为 API 授权与字段语义说明源。
3. 公开抓取阶段只下钻 public protocol 页面，不伪造 Bearer token 调私有 API。
4. 从 protocol 页面抽取 `protocol -> version -> step`、materials、references 和 DOI。
5. 附件与下载资源单独记为 asset 层，不和正文 step 混写。
爬取脚本
from pathlib import Path
import re
import sys

sys.path.append(str(Path(__file__).resolve().parents[1]))
from common import fetch_text, save_snapshot


def main() -> None:
    robots = fetch_text("https://www.protocols.io/robots.txt")
    dev = fetch_text("https://www.protocols.io/developers")
    payload = {
        "api_disallow_lines": [line for line in robots.get("text", "").splitlines() if "Disallow:" in line and "/api/" in line],
        "developer_page_has_oauth": bool(re.search(r"OAuth|Bearer", dev.get("text", ""), re.I)),
        "developer_page_preview": dev.get("text", "")[:300],
    }
    print(save_snapshot("wet/protocols_io.json", payload))


if __name__ == "__main__":
    main()
