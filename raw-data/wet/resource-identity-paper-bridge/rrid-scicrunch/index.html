RRID / SciCrunch

Resource Type

资源身份解析平台。

Unit

RRID record

完整内容

资源类型分类

资源身份解析平台。

展示层级

P0 资源身份层。

在湿实验资源图谱中的定位

用于把抗体、工具、数据库、模型生物等资源映射到稳定标识。

推荐入库单元

RRID record

推荐结构表

resource_identity、resource_synonym

数据来源与覆盖领域

来自 rrids.org 官方说明站与 SciCrunch 聚合索引体系。官方说明页明确写明 The RRID portal displays the aggregated RRID index，并将抗体、细胞系、质粒等资源分别挂到各自权威来源。

数据规模与更新情况

rrids.org sitemap 当前可见 46 个公开内容页。说明页还明确写出 index updates 由源数据更新驱动，通常会有数天的索引滞后；真正的聚合记录入口是 scicrunch.org/resources 与 rrid.site，但当前环境对该层访问受限。

下载单元数量

若按当前稳定可抓的公开说明层统计，rrids.org sitemap 可见 46 个内容页；RRID record 级总量因聚合入口受限未写入硬数字。

单个下载单元平均大小

按 5 个公开说明页抽样，平均约 90.68 KB。

数据格式与字段结构

说明页把 RRID 描述为聚合自权威源的 index。可落成 RRID → resource record → provider / citation / metadata / status。资源来源映射也相当明确，例如 antibodies → Antibody Registry、cell lines → Cellosaurus、plasmids → Addgene。

数据内容特点与适用场景

最适合做 protocol 中抗体、细胞系、质粒、软件、数据库和模型资源的稳定标识回连，以及多资源来源之间的 authority mapping。

获取方式、开放性与可用性

说明页、规则页和 sitemap 可公开访问；聚合检索层和更深的 SciCrunch 资源页当前环境受限，因此更适合先接 RRID 解析规则与 authority mapping。

准入判断

公开说明页抓取成功，SciCrunch 聚合索引入口当前环境受限。

若后续接入，建议定位键

RRID

公开入口与发现入口

  • https://www.rrids.org/rrid-system
  • https://scicrunch.org/resources

官方对象结构与实际下载单元

  • RRID → resource record
  • resource record → provider / citation / metadata / status
  • resource type → authority(Antibody Registry / Cellosaurus / Addgene / BioSamples authority)

可以确定的文件格式

HTML

是否有不确定的文件格式

有

  • 聚合索引层的返回格式与 API 能力依 SciCrunch developer tools 而变

单平台复核结论

说明页已经足够把 RRID 的 authority mapping 讲清楚；真正的大规模 record 抓取仍要等聚合检索层访问条件更稳定。
