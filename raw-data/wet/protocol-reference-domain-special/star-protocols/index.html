Item 4.1

STAR Protocols

Resource Type

模板化方法文章平台。

Unit

article

完整内容

资源类型分类

模板化方法文章平台。

展示层级

P1 强参考层。

在湿实验资源图谱中的定位

适合作为结构 benchmark 与 schema 对齐参考，尤其适合 key resources 与 troubleshooting 段落建模。

推荐入库单元

article

推荐结构表

reference_article、article_section、resource_table

数据来源与覆盖领域

来自 STAR Protocols 官方期刊、ScienceDirect article 页与作者指南。官方长期强调 step-by-step methods、key resources table、expected outcomes 与 troubleshooting 等模板化写法。

数据规模与更新情况

当前可直接访问 ScienceDirect 的 journal issues 与 article 页。代表性 article 页的预加载 JSON 中公开暴露 publicationOpenAccess.oaArticleCount = 4369；最新可见的 Volume 7, Issue 2 页面可枚举 44 个 PII article URL。

下载单元数量

若按 journal 公开 article 口径统计，当前可复核 4,369 个 open-access articles；若按最新 issue 样例层统计，Volume 7, Issue 2 可见 44 个 PII article URL。

单个下载单元平均大小

按最新 issue 前 5 个 article 页抽样，平均约 179.47 KB。

数据格式与字段结构

article 页同时暴露 HTML 与预加载 JSON；公开字段可直接观察到 Summary、Graphical abstract、Highlights、Subject areas、DOI、卷期、开放许可与 PDF 入口。作者指南另外定义了 Key resources table、method details、expected outcomes、troubleshooting 等结构锚点。

数据内容特点与适用场景

非常适合做 protocol 模板建模、资源表抽取、图文摘要对齐和 troubleshooting 段落抽取。

获取方式、开放性与可用性

cell.com 的期刊首页与作者指南可复核；ScienceDirect 的 issue 与 article 页当前环境可直接访问。/search/api 需要 token，当前环境返回 401 或访问校验，不能作为稳定批量主入口。

准入判断

issue 与 article 页抓取成功；搜索 API 当前不稳定。

若后续接入，建议定位键

PII / DOI / article URL

公开入口与发现入口

  • https://www.sciencedirect.com/journal/star-protocols/issues
  • https://www.sciencedirect.com/science/article/pii/S2666166726000778

官方对象结构与实际下载单元

  • journal → issue → article(PII/DOI)
  • article → summary / highlights / graphical abstract / subject areas
  • article → PDF link / open license / metadata JSON

可以确定的文件格式

HTML、JSON(preloaded state)、PDF link

是否有不确定的文件格式

有

  • 图像与补充材料类型依具体 article 而变

单平台复核结论

正文 article 层已经可见，推荐以 issue TOC 枚举 PII，再下钻 article 页；搜索 API 暂不作为稳定入口。

爬取脚本
from pathlib import Path
import sys

sys.path.append(str(Path(__file__).resolve().parents[1]))
from common import fetch_text, preview_text, save_snapshot


URLS = [{'label': 'robots 规则', 'url': 'https://www.cell.com/robots.txt'}, {'label': 'sitemap 探测', 'url': 'https://www.cell.com/sitemap-index-1.txt'}, {'label': '期刊主页探测', 'url': 'https://www.cell.com/star-protocols/home'}]


def main() -> None:
    records = []
    for item in URLS:
        response = fetch_text(item["url"])
        records.append(
            {
                "label": item["label"],
                "url": item["url"],
                "status": response.get("status"),
                "final_url": response.get("final_url"),
                "content_type": response.get("content_type"),
                "preview": preview_text(response.get("text", "")),
            }
        )
    path = save_snapshot("wet/star_protocols.json", {"slug": "star_protocols", "records": records})
    print(path)


if __name__ == "__main__":
    main()
