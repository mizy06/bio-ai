ENCODE

Resource Type

实验数据仓库 / 规范化功能基因组平台。

Unit

以 Experiment / File / Biosample / Donor 为主，批量侧优先入 metadata.tsv、JSON 搜索结果和文件清单。

Primary Key

ENCODE accession（experiment / file / biosample）

完整内容

资源类型分类

实验数据仓库 / 规范化功能基因组平台。

展示层级

P0 主库核心源。

在干实验资源图谱中的定位

适合作为元数据质量较高、规范化程度较强的功能基因组学样板库；对训练“会选文件、会读 QC”的分析代理很有价值。

推荐入库单元

以 Experiment / File / Biosample / Donor 为主，批量侧优先入 metadata.tsv、JSON 搜索结果和文件清单。

推荐结构表

encode_experiment、encode_file、encode_biosample、encode_donor、encode_replicate、encode_library。

数据来源与覆盖领域

ENCODE Project Portal，覆盖 功能基因组 / 调控组；可提供 ChIP-seq、ATAC-seq、RNA-seq、single-cell 等实验与统一分析产物。

数据规模与更新情况

官方搜索接口当前可核实约 27,967 个 experiments、30,497 个 biosamples、1,562,418 个 files；属于长期持续扩展型资源。

下载单元数量

按推荐下载单元中的 File 口径，当前可核实约 1,562,418 个文件记录。

单个下载单元平均大小

按官方 File 搜索结果中的 file_size 字段抽样 5 个 file 计算，平均约 1.59 MB。

数据格式与字段结构

典型数据形态为 raw + processed files + JSON metadata；官方对象主轴可概括为 Experiment → Replicate → Library → File；Biosample → Donor；Experiment → File → derived_from；主要消费格式可概括为 官方 search 可枚举：bed、bigWig、fastq、bam、bigBed、tsv、tar、txt、bedpe、bigInteract、hic、starch、tagAlign、gtf、sra、gff、idat、hdf5、pairs、rcc、sam、wig、fasta、vcf、csv、cndb、csfasta、h5ad、csqual、CEL、idx。

数据内容特点与适用场景

最适合 功能基因组标准流程学习、质量控制、peak/表达分析。为什么值得补：元数据和质量标准都很强，适合训练“规范化分析 agent”。主要提醒：对象模型较复杂，需要先做 metadata 映射。

获取方式、开放性与可用性

开放；程序化访问方式为 Portal / metadata / structured JSON。实际抓取时建议优先走官方批量入口，不要用页面抓取代替结构化接口。

准入判断

批量抓取成功；批量枚举适合优先使用 JSON 搜索、metadata.tsv 和 batch_download，不宜用页面级抓取替代。

推荐批量主键

ENCODE accession（experiment / file / biosample）

暂无

最小爬取伪代码

1. 从 `search/?type=Experiment&format=json` 开始枚举实验级 JSON 记录。
2. 同步抓取 `metadata.tsv` 与 `batch_download`，核对实验到文件的映射。
3. 从实验 JSON 中抽取 accession、assay、biosample、file 列表。
4. 对文件层再按 `file accession` 保留下载 URL、格式、assembly 和 output type。
5. 最终分别沉淀 `experiment`、`biosample`、`file` 三层对象。
爬取脚本
from pathlib import Path
import sys

sys.path.append(str(Path(__file__).resolve().parents[1]))
from common import fetch_json, save_snapshot


def main() -> None:
    experiments = fetch_json("https://www.encodeproject.org/search/?type=Experiment&format=json&limit=1")
    payload = {
        "sample_accessions": [item.get("accession") for item in (experiments.get("json") or {}).get("@graph", [])],
        "status": experiments.get("status"),
        "content_type": experiments.get("content_type"),
    }
    print(save_snapshot("dry/encode.json", payload))


if __name__ == "__main__":
    main()
