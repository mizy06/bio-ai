Item 2.2

SRA

Resource Type

实验数据仓库 / 原始测序归档。

Unit

以 SRP / SRX / SRR 层级记录和 Run 级元数据表为主，原始文件通过官方工具按需拉取，不建议直接镜像全量 FASTQ。

Primary Key

SRP / SRX / SRR accession 或 BioProject accession

完整内容

资源类型分类

实验数据仓库 / 原始测序归档。

展示层级

P0 主库核心源。

在干实验资源图谱中的定位

适合作为测序类干实验任务最底层、最接近 raw 的公共入口；对重跑 RNA-seq、WGS/WES、amplicon 等流程尤其关键。

推荐入库单元

以 SRP / SRX / SRR 层级记录和 Run 级元数据表为主，原始文件通过官方工具按需拉取，不建议直接镜像全量 FASTQ。

推荐结构表

sra_study 主表、sra_experiment 实验表、sra_run 运行表、sra_biosample_link、sra_bioproject_link、sra_file_asset。

数据来源与覆盖领域

NCBI Sequence Read Archive，覆盖 原始测序数据；可提供 raw reads、alignment info、项目级元数据。

数据规模与更新情况

属于超大规模、持续更新的原始测序归档；更适合按项目、experiment、run 维度检索与调度，而不是依赖一个静态全站总量。

数据格式与字段结构

典型数据形态为 FASTQ/BAM + metadata；官方对象主轴可概括为 Study(SRP) → Experiment(SRX) → Run(SRR)；BioProject → BioSample → Experiment(SRX) → Run(SRR)；Run(SRR) → metadata table / manifest / download；主要消费格式可概括为 闭合部分：SRA、FASTQ；不闭合部分：部分项目还会挂接其他分析文件，官方未给全站封闭格式全集。

数据内容特点与适用场景

最适合 从原始数据重跑 RNA-seq / WGS / WES / amplicon 分析流程。为什么值得补：做‘analysis agent’时最接近真实复现任务。主要提醒：下载和计算成本高；需重点处理样本信息与 QC。

获取方式、开放性与可用性

开放为主；程序化访问方式为 NCBI 下载/云访问。实际抓取时建议优先走官方批量入口，不要用页面抓取代替结构化接口。

下载单元数量

若按官方 db=sra 记录口径统计，当前全库可核约 43,560,760 条记录；后续清洗入库时仍建议把主键拆回 SRP / SRX / SRR 层级。

单个下载单元平均大小

按 5 条近期单-run 记录的 total_size 字段抽样计算，平均约 232.03 MB；该值对应记录级归档大小，不代表所有 SRR 的真实全局平均。

准入判断

批量抓取成功；优先入口适合选择 Entrez / Run Selector / SRA Toolkit，原始文件下载不宜依赖网页 HTML 抓取。

推荐批量主键

SRP / SRX / SRR accession 或 BioProject accession

公开入口与发现入口

暂无

爬取脚本
from pathlib import Path
import sys

sys.path.append(str(Path(__file__).resolve().parents[1]))
from common import fetch_json, save_snapshot


def main() -> None:
    search = fetch_json("https://eutils.ncbi.nlm.nih.gov/entrez/eutils/esearch.fcgi?db=sra&term=bioproject&retmax=5&retmode=json")
    payload = {
        "count": ((search.get("json") or {}).get("esearchresult") or {}).get("count"),
        "idlist": ((search.get("json") or {}).get("esearchresult") or {}).get("idlist"),
    }
    print(save_snapshot("dry/sra.json", payload))


if __name__ == "__main__":
    main()
