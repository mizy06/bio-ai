GEO

Resource Type

实验数据仓库 / 功能基因组学公共库。

Unit

以 GSE / GSM / GPL / GDS 记录为主；结构化文件优先入 Series Matrix / MINiML / SOFT，supplementary files 先保留清单与链接。

Primary Key

GSE / GSM / GPL / GDS accession

完整内容

资源类型分类

实验数据仓库 / 功能基因组学公共库。

展示层级

P0 主库核心源。

在干实验资源图谱中的定位

适合作为表达组学与功能基因组学的主入库源，同时保留项目层（GSE）和样本层（GSM），方便任务筛选、样本检索和下载调度。

推荐入库单元

以 GSE / GSM / GPL / GDS 记录为主；结构化文件优先入 Series Matrix / MINiML / SOFT，supplementary files 先保留清单与链接。

推荐结构表

geo_series 主表、geo_sample 子表、geo_platform 平台表、geo_dataset 派生表、geo_series_sample_map 关联表、geo_file_asset 文件表。

数据来源与覆盖领域

NCBI Gene Expression Omnibus，覆盖 转录组 / 功能基因组；可提供 Series、Sample、Platform、processed data、部分差异表达与元数据。

数据规模与更新情况

属于持续滚动更新型公共库；官方 E-utilities 的 gds 口径是混合库，不能直接当“数据集总数”，后续清洗必须按 GSE/GSM/GPL/GDS 前缀再分层。

数据格式与字段结构

典型数据形态为 表型+表达矩阵+元数据；官方对象主轴可概括为 gds(Entrez db) → GSE / GSM / GPL / GDS；GSE → linked GSM / GPL；GSE → Series Matrix / MINiML / SOFT；主要消费格式可概括为 闭合部分：SOFT、MINiML(XML)、Series Matrix(TXT/GZ)；不闭合部分：supplementary files 官方未定义全站封闭格式全集。

数据内容特点与适用场景

最适合 课题检索、复现公开分析、差异表达/聚类/富集分析 benchmarking。为什么值得补：最常见的 dry-lab 入门数据源之一；和文献联动强。主要提醒：processed data 质量参差不齐；不同平台需统一预处理。

获取方式、开放性与可用性

开放；程序化访问方式为 NCBI 接口/下载通道。实际抓取时建议优先走官方批量入口，不要用页面抓取代替结构化接口。

下载单元数量

按推荐下载单元中的 GSE 口径，当前通过 NCBI E-utilities 可核约 280,804 个 Series。

单个下载单元平均大小

按 5 个 GSE 的 Series Matrix 文件抽样计算，平均约 3.44 KB；这只代表结构化矩阵文件，不代表 supplementary files 总大小。

准入判断

批量抓取成功；元数据批量检索适合走 Entrez E-utilities，结构化文件适合走 GEO 官方 HTTP/FTP 目录，supplementary files 仍需按 accession 枚举，不宜假设统一 schema。

推荐批量主键

GSE / GSM / GPL / GDS accession

暂无

最小爬取伪代码

1. 调用 Entrez ESearch(`db=gds`) 先枚举 `gds` 库中的 GEO 记录。
2. 按 accession 前缀把返回结果拆分成 `GSE / GSM / GPL / GDS` 四类对象。
3. 对 `GSE` 再下钻到 `Series Matrix / MINiML / SOFT` 官方文件。
4. 对 `GSM / GPL` 只补抓对应详情页和 supplementary file 链接。
5. 把 accession、对象类型、详情链接和下载链接分别落表，而不是混成单层记录。
爬取脚本
from pathlib import Path
import sys

sys.path.append(str(Path(__file__).resolve().parents[1]))
from common import fetch_json, fetch_text, save_snapshot


def main() -> None:
    search = fetch_json("https://eutils.ncbi.nlm.nih.gov/entrez/eutils/esearch.fcgi?db=gds&term=gse[ETYP]&retmax=5&retmode=json")
    readme = fetch_text("https://ftp.ncbi.nlm.nih.gov/geo/README.txt")
    payload = {
        "search_count": ((search.get("json") or {}).get("esearchresult") or {}).get("count"),
        "sample_ids": ((search.get("json") or {}).get("esearchresult") or {}).get("idlist"),
        "ftp_readme_preview": readme.get("text", "")[:300],
    }
    print(save_snapshot("dry/geo.json", payload))


if __name__ == "__main__":
    main()
