完整内容

资源类型分类

实验数据仓库 / 癌症多组学公共+受控平台。

展示层级

P0 主库核心源。

在干实验资源图谱中的定位

适合作为癌症多组学主库；公开层可直接进入核心数据层，受控层更适合保留元数据、manifest 和授权状态，而不直接镜像受控文件。

推荐入库单元

以 project / case / sample / file 关系图为主，下载侧优先保留 manifest、file metadata 和可公开文件链接。

推荐结构表

gdc_project、gdc_case、gdc_sample、gdc_aliquot、gdc_file、gdc_clinical_event、gdc_manifest。

数据来源与覆盖领域

NCI Genomic Data Commons，覆盖 癌症多组学；可提供 标准化临床、基因组、转录组、表观组、蛋白组数据。

数据规模与更新情况

官方 API 当前可核实约 91 个 projects、50,270 个 cases、1,271,450 个 files；状态接口显示公开版本为 Data Release 45.0，发布日期 2025-12-04。

下载单元数量

按推荐下载单元中的 file 口径，当前可核实约 1,271,450 个公开文件记录。

单个下载单元平均大小

按官方 file_size 字段抽样 5 个 file 计算，平均约 5.32 GB。

数据格式与字段结构

典型数据形态为 clinical + omics matrices/files；官方对象主轴可概括为 project → case → sample → portion → analyte → aliquot → file；project → case → diagnosis / treatment / follow_up；file → index / metadata / download / manifest；主要消费格式可概括为 目前官方可枚举主集合：BAM、BAI、BCF、BEDPE、CEL、CRAM、CRAI、CSV、IDAT、JSON、MAF、PDF、SVS、TAR、TSV、TXT、VCF、XML。

数据内容特点与适用场景

最适合 癌症 cohort 构建、多组学关联分析、预后/亚型研究。为什么值得补：标准化程度高，适合做可复现分析 agent。主要提醒：受控数据访问合规要求高；任务设计要区分 open vs controlled。

获取方式、开放性与可用性

开放与受控并存；程序化访问方式为 Portal + API + transfer tool。实际抓取时建议优先走官方批量入口，不要用页面抓取代替结构化接口。

准入判断

本轮公开部分抓取成功，受控部分必须走授权 token 和官方数据传输通道；不应抓网页代替 API / manifest。

爬取脚本
from pathlib import Path
import sys

sys.path.append(str(Path(__file__).resolve().parents[1]))
from common import fetch_json, save_snapshot


def main() -> None:
    status = fetch_json("https://api.gdc.cancer.gov/status")
    files = fetch_json("https://api.gdc.cancer.gov/files?size=3&format=JSON")
    hits = ((files.get("json") or {}).get("data") or {}).get("hits") or []
    payload = {
        "release": (status.get("json") or {}).get("data_release"),
        "sample_files": [
            {
                "id": hit.get("id"),
                "file_name": hit.get("file_name"),
                "data_type": hit.get("data_type"),
                "access": hit.get("access"),
            }
            for hit in hits
        ],
    }
    print(save_snapshot("dry/gdc_tcga.json", payload))


if __name__ == "__main__":
    main()
