在干实验资源图谱中的定位

适合作为大体量质谱 raw 文件与再分析任务的主入口；它对“下载-解包-重跑”型分析流程特别有价值。

推荐入库单元

以 dataset(MSV) 为主，优先保留 dataset metadata、列表页统计字段和文件清单入口。

推荐结构表

massive_dataset、massive_stat_snapshot、massive_file_asset、massive_reanalysis_link、massive_publication_link。

数据来源与覆盖领域

MassIVE Repository，覆盖 质谱 / 蛋白组 / 代谢组相关；可提供 大规模质谱数据、再分析与知识库入口。

数据规模与更新情况

属于持续更新的大型质谱仓库；列表页稳定提供 dataset 级文件数、大小和 reanalysis 统计，适合先做目录层入库。

数据格式与字段结构

典型数据形态为 mass spectrometry raw/result files；官方对象主轴可概括为 dataset → file list；dataset → quantitative analysis；dataset → reanalysis containers；主要消费格式可概括为 不闭合：官方未公布全站封闭文件格式全集；可稳定抽取的是 dataset 级统计字段，不是完整文件后缀全集。

数据内容特点与适用场景

最适合 质谱数据再分析、谱图库相关任务。为什么值得补：适合扩展到更真实的 MS 工作流。主要提醒：数据与工具门槛更高，建议第二阶段接入。

获取方式、开放性与可用性

开放；程序化访问方式为 站点与下载接口。实际抓取时建议优先走官方批量入口，不要用页面抓取代替结构化接口。

下载单元数量

按推荐下载单元中的 MSV dataset 口径，当前公开列表可核约 19,177 个 dataset。

单个下载单元平均大小

按 5 个 MSV dataset 的列表级总大小字段抽样计算，平均约 18.63 GB。

准入判断

批量抓取成功；当前稳定入口是官方 dataset 列表页；站内 AJAX QueryDatasets 虽存在，但公开参数契约尚未确认，不宜在无额外验证时直接依赖。

爬取脚本
from pathlib import Path
import sys

sys.path.append(str(Path(__file__).resolve().parents[1]))
from common import fetch_text, preview_text, save_snapshot


URLS = [{'label': 'dataset 列表页', 'url': 'https://massive.ucsd.edu/ProteoSAFe/datasets.jsp'}, {'label': '未带参数探测 `QueryDatasets`', 'url': 'https://massive.ucsd.edu/ProteoSAFe/QueryDatasets?target=datasets'}]


def main() -> None:
    records = []
    for item in URLS:
        response = fetch_text(item["url"])
        records.append(
            {
                "label": item["label"],
                "url": item["url"],
                "status": response.get("status"),
                "final_url": response.get("final_url"),
                "content_type": response.get("content_type"),
                "preview": preview_text(response.get("text", "")),
            }
        )
    path = save_snapshot("dry/massive.json", {"slug": "massive", "records": records})
    print(path)


if __name__ == "__main__":
    main()
