MetaboLights

适合作为代谢组与 ISA 风格元数据的主库，能把干实验 agent 的多组学覆盖从转录组/蛋白组补到代谢组。

完整内容

资源类型分类

建议纳入层级

P0 主库核心源。

对干实验 agent raw 数据库的定位

适合作为代谢组与 ISA 风格元数据的主库，能把干实验 agent 的多组学覆盖从转录组/蛋白组补到代谢组。

推荐入库单元

以 study(MTBLS) 为主，保留 investigation、sample、assay、protocol 和 data files。

建议拆表

mtbls_study、mtbls_investigation、mtbls_sample、mtbls_assay、mtbls_protocol、mtbls_file_asset。

数据来源与覆盖领域

EMBL-EBI MetaboLights，覆盖 代谢组；可提供 代谢组 studies、raw data、metadata、compound 信息。

数据规模与更新情况

官方 Web Service 当前可核实约 2,833 个 studies；属于持续收录型仓库，和 ISA 元数据结构结合紧密。

下载单元数量

按推荐下载单元中的 MTBLS study 口径，当前可核约 2,833 个 study。

单个下载单元平均大小

按 5 个公开 MTBLS study 的官方 FTP 目录递归文件总大小抽样计算，平均约 3.96 GB。

数据格式与字段结构

典型数据形态为 study packages + metadata；官方对象主轴可概括为 study → investigation；study → sample；study → assay；主要消费格式可概括为 闭合部分：ISA-Tab、ISA-JSON；不闭合部分：raw/derived assay files 官方未给全站封闭全集。

数据内容特点与适用场景

最适合 代谢组数据检索、标准流程复现、跨 study 比较。为什么值得补：补齐代谢组赛道，避免只盯转录组/单细胞。主要提醒：不同仪器与实验设计差异大，标准化成本高。

获取方式、开放性与可用性

开放；程序化访问方式为 公开下载。实际抓取时建议优先走官方批量入口，不要用页面抓取代替结构化接口。

准入判断

本轮抓取成功；应优先走官方 Web Service 与 study 下载，而不是 HTML 抓取。

爬取脚本
from pathlib import Path
import sys

sys.path.append(str(Path(__file__).resolve().parents[1]))
from common import fetch_json, save_snapshot


def main() -> None:
    studies = fetch_json("https://www.ebi.ac.uk/metabolights/ws/studies")
    detail = fetch_json("https://www.ebi.ac.uk/metabolights/ws/studies/MTBLS1")
    content = (detail.get("json") or {}).get("mtblsStudy") or {}
    payload = {
        "study_list_size": len((studies.get("json") or {}).get("content") or []),
        "detail_accession": content.get("accession"),
        "study_status": content.get("studyStatus"),
        "technology": content.get("studyDesignDescriptors"),
    }
    print(save_snapshot("dry/metabolights.json", payload))


if __name__ == "__main__":
    main()
