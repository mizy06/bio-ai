Item 2.6

PRIDE

Resource Type

实验数据仓库 / 蛋白组归档。

Unit

以 project(PXD) 为主，重点入 project metadata、file list、sample attributes 和 SDRF。

Primary Key

PXD accession

完整内容

资源类型分类

实验数据仓库 / 蛋白组归档。

展示层级

P0 主库核心源。

在干实验资源图谱中的定位

适合作为蛋白组 raw 数据、结果文件和样本注释的主入口之一；对蛋白组复现和 reanalysis 都很重要。

推荐入库单元

以 project(PXD) 为主，重点入 project metadata、file list、sample attributes 和 SDRF。

推荐结构表

pride_project、pride_file、pride_sample_attribute、pride_sdrf、pride_publication_link。

数据来源与覆盖领域

PRIDE Archive，覆盖 蛋白组；可提供 质谱蛋白组公共数据、鉴定结果、项目元数据。

数据规模与更新情况

属于持续快速增长的主流蛋白组库；官方提供项目 API，但更适合按 accession 检索，而不是使用单一静态总量。

数据格式与字段结构

典型数据形态为 raw / mzML / result files + metadata；官方对象主轴可概括为 project → project metadata；project → file list；project → sample attributes；主要消费格式可概括为 闭合核心标准：mzML、mzIdentML、mzTab；不闭合部分：原始厂商文件和补充文件官方未给全站封闭全集。

数据内容特点与适用场景

最适合 蛋白鉴定复现、蛋白组再分析、公开结果验证。为什么值得补：蛋白组方向最常用公开库之一。主要提醒：文件大、格式复杂，分析流程门槛较高。

获取方式、开放性与可用性

开放；程序化访问方式为 PRIDE API / download。实际抓取时建议优先走官方批量入口，不要用页面抓取代替结构化接口。

下载单元数量

按推荐下载单元中的 PXD project 口径，当前通过 PRIDE Archive API 可核约 37,876 个 project。

单个下载单元平均大小

按 5 个 PXD project 的项目级 fileSizeBytes 汇总抽样计算，平均约 10.75 GB。

准入判断

批量抓取成功；优先入口适合选择 PRIDE Archive Web Service，而不是页面抓取。

PXD accession

爬取脚本
from pathlib import Path
import sys

sys.path.append(str(Path(__file__).resolve().parents[1]))
from common import fetch_json, save_snapshot


def main() -> None:
    page = fetch_json("https://www.ebi.ac.uk/pride/ws/archive/v2/projects?pageSize=3&page=0")
    detail = fetch_json("https://www.ebi.ac.uk/pride/ws/archive/v2/projects/PXD001357")
    payload = {
        "projects": [
            {"accession": item.get("accession"), "title": item.get("title")}
            for item in (page.get("json") or [])
        ],
        "detail_accession": (detail.get("json") or {}).get("accession"),
        "detail_species": (detail.get("json") or {}).get("species"),
    }
    print(save_snapshot("dry/pride.json", payload))


if __name__ == "__main__":
    main()
