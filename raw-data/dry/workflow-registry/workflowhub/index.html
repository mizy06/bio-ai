WorkflowHub

Resource Type

Workflow registry / RO-Crate 工作流枢纽。

Unit

以 workflow / version / RO-Crate 为主，保留版本、作者、license、descriptor 和 crate 资产。

Primary Key

workflow id / version id / TRS tool id

完整内容

资源类型分类

Workflow registry / RO-Crate 工作流枢纽。

展示层级

P0 workflow 定义主层。

在干实验资源图谱中的定位

适合作为 workflow metadata 与版本关系的主层，尤其适合提供“先看流程定义、再抓 descriptor”的入口。

推荐入库单元

以 workflow / version / RO-Crate 为主，保留版本、作者、license、descriptor 和 crate 资产。

推荐结构表

workflowhub_workflow、workflowhub_version、workflowhub_rocrate、workflowhub_file_asset、workflowhub_author。

数据来源与覆盖领域

Workflow 注册表，覆盖范围为 Dry；核心用途是 workflow 元数据、RO-Crate、可复用计算流程检索。

数据规模与更新情况

当前状态 Active；平台更适合作为“核心保留”层，通常不公开统一稳定总量。备注：这是 dry-lab workflow 语料的核心入口之一，应从 protocol 类资源中独立出来。

数据格式与字段结构

核心对象可概括为 workflow → version；workflow → RO-Crate；workflow → metadata / authors / license；主要消费格式可概括为 闭合部分：RO-Crate；不闭合部分：具体 workflow 文件格式取决于被收录工作流语言。

数据内容特点与适用场景

适合 workflow 元数据、RO-Crate、可复用计算流程检索；protocol 生成适配度 低-中，dry-lab workflow 生成适配度 高，可直接作为分析数据源：否。

获取方式、开放性与可用性

开放性为 开放；程序化访问情况：可下载工作流与元数据。实际抓取边界以下方“准入判断”为准。

下载单元数量

按推荐下载单元中的 TRS tool/workflow 口径，当前通过 GA4GH TRS 接口可枚举约 1,461 条记录。

单个下载单元平均大小

按 5 个 workflow descriptor 抽样计算，平均约 27.28 KB。

准入判断

批量抓取成功；GA4GH TRS 是稳定的批量入口，网页抓取更适合作为补充。
