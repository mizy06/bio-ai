Item 3.1

cBioPortal

Resource Type

整理型分析门户 / 派生队列数据层。

Unit

以 study / patient / sample / molecularProfile 为主，重点入临床表、分子事件表和 profile 级下载资产。

Primary Key

studyId + patientId / sampleId / molecularProfileId

完整内容

资源类型分类

整理型分析门户 / 派生队列数据层。

展示层级

P1 主库补充层。

在干实验资源图谱中的定位

更适合作为标准化 cohort 检索和快速分析入口，而不是原始事实源；可提供“已整理好的癌症队列视图”。

推荐入库单元

以 study / patient / sample / molecularProfile 为主，重点入临床表、分子事件表和 profile 级下载资产。

推荐结构表

cbio_study、cbio_patient、cbio_sample、cbio_molecular_profile、cbio_clinical_record、cbio_molecular_event。

数据来源与覆盖领域

cBioPortal for Cancer Genomics，覆盖 癌症 genomics 可视化/分析门户；可提供 整理后的癌症 genomics studies、可视化和下载接口。

数据规模与更新情况

官方公开 API 当前可核实约 535 个 public studies；平台属于持续导入型门户，适合作为整理后 cohort 视图层。

下载单元数量

按推荐下载单元中的 study 口径，当前可核约 535 个 public study。

单个下载单元平均大小

按官方 study 列表接口抽样 5 个 study 元数据记录计算，平均约 500 B；这反映的是 API 记录体量，不代表全部下游数据文件体积。

数据格式与字段结构

典型数据形态为 study-level tables / portal API；官方对象主轴可概括为 study → patient → sample；study → sampleList → molecularProfile；study → clinical(patient / sample)；主要消费格式可概括为 消费层闭合：JSON(API)、TSV(download)、MAF(部分分子事件下载)；导入层不是单一闭合集。

数据内容特点与适用场景

最适合 快速队列探索、突变/拷贝数/临床关联分析。为什么值得补：适合做快速交互分析与报告生成，不必总从原始文件起步。主要提醒：更像分析门户而非原始数据总库。

获取方式、开放性与可用性

开放；程序化访问方式为 REST API。实际抓取时建议优先走官方批量入口，不要用页面抓取代替结构化接口。

准入判断

批量抓取成功；适合直接使用 Web API，不建议回退到网页抓取。

推荐批量主键

studyId + patientId / sampleId / molecularProfileId

公开入口与发现入口

暂无

爬取脚本
from pathlib import Path
import sys

sys.path.append(str(Path(__file__).resolve().parents[1]))
from common import fetch_json, save_snapshot


def main() -> None:
    info = fetch_json("https://www.cbioportal.org/api/info")
    studies = fetch_json("https://www.cbioportal.org/api/studies?pageSize=3&pageNumber=0&projection=SUMMARY")
    payload = {
        "portal_version": (info.get("json") or {}).get("portalVersion"),
        "studies": [
            {"studyId": item.get("studyId"), "name": item.get("name"), "cancerTypeId": item.get("cancerTypeId")}
            for item in (studies.get("json") or [])
        ],
    }
    print(save_snapshot("dry/cbioportal.json", payload))


if __name__ == "__main__":
    main()
