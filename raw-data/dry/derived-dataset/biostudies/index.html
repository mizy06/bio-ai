数据来源与覆盖领域

EMBL-EBI BioStudies，覆盖 study-level 打包 / 多组学；可提供 study 描述、补充文件、指向其他 EBI 资源的链接。

数据规模与更新情况

当前搜索 API 可核约 80,947 个 study 命中；资源强调“研究单元”和跨资源链接，而不是统一首页总量。

下载单元数量

按推荐下载单元中的 study accession 口径，当前可核约 80,947 个 study。

单个下载单元平均大小

按搜索 API 抽样 5 个 study 命中的元数据记录计算，平均约 287 B；这只是索引命中记录大小，不代表 study 附件总大小。

数据格式与字段结构

典型数据形态为 study metadata + linked assets；官方对象主轴可概括为 study → attributes；study → section → subsection；study → files；主要消费格式可概括为 不闭合：平台本身就是研究级文件打包层，官方不定义封闭文件格式全集。

数据内容特点与适用场景

最适合 围绕一篇研究整理全部相关数据资产；多模态 study 打包。为什么值得补：适合未来做‘按 study 组织的全流程 agent’。主要提醒：它更多是包装层，不一定直接给标准化矩阵。

获取方式、开放性与可用性

开放；程序化访问方式为 门户与下载。实际抓取时建议优先走官方批量入口，不要用页面抓取代替结构化接口。

最小爬取伪代码

1. 先通过 `search` 接口分页枚举 study accession。
2. 对每个 accession 请求 study 详情与 `info` 文件信息接口。
3. 从详情里拆出 section、attribute、link、file table。
4. 把 `study metadata` 和 `file listing` 分表保存。
5. 搜索页只负责发现 accession，结构化内容以 detail/info 为准。
爬取脚本
from pathlib import Path
import sys

sys.path.append(str(Path(__file__).resolve().parents[1]))
from common import fetch_json, save_snapshot


def main() -> None:
    search = fetch_json("https://www.ebi.ac.uk/biostudies/api/v1/search?query=*:*&pageSize=2&page=1")
    detail = fetch_json("https://www.ebi.ac.uk/biostudies/api/v1/studies/S-BSST1")
    info = fetch_json("https://www.ebi.ac.uk/biostudies/api/v1/studies/S-BSST1/info")
    payload = {
        "total_hits": (search.get("json") or {}).get("totalHits"),
        "sample_accessions": [item.get("accession") for item in (search.get("json") or {}).get("hits", [])],
        "detail_accno": (detail.get("json") or {}).get("accno"),
        "file_columns": [col.get("name") for col in (info.get("json") or {}).get("columns", [])],
    }
    print(save_snapshot("dry/biostudies.json", payload))


if __name__ == "__main__":
    main()
